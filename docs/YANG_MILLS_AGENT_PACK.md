# Yang–Mills Agent Integration

The core graph is broad by design. The Yang–Mills integration is a domain pack:
it stores a live theorem frontier, not a replacement ontology.

## Design goals

1. **Freshness:** every recommendation is tied to an inspected repository HEAD.
2. **Status discipline:** verified and planned declarations are distinct values.
3. **Semantic routing:** producers and consumers are linked through named predicates.
4. **Branch fidelity:** geometric and marginal scale mechanisms never share an unsuffixed theorem name.
5. **Information preservation:** support, target, measure and physical identity are tracked until their consumers fire.
6. **Source honesty:** papers are anchors for statements, not substitutes for Lean proofs.
7. **Uniformity:** constant dependencies and quantifier order are machine-visible.
8. **Executable handoff:** every proposed commit has verification targets.

## Recommended agent loop

```text
refresh HEAD
→ query current declaration registry
→ find the shortest unresolved route
→ audit forgotten information
→ choose geometric or marginal branch
→ expose every remaining hypothesis
→ propose exact theorem/file names
→ include verification commands
→ update the pack after verification
```

## Quality definition

“100%” here means that all ten pack gates and all ten 10-point evaluation
scenarios pass. It does not mean the Yang–Mills mathematics is complete.
